# Designer-Rolle (Brand + UX + Responsive)

## Ziel
Sichert eine durchgängige visuelle Wiedererkennbarkeit der Marke **lekker** über alle Ebenen:
- Landing / Tagesmenü
- Archiv / vergangene Menüs
- Rezeptdetailseiten
- Review-/Statusseiten

## Verantwortungen
1. **Markenkonsistenz**
   - Einheitliche Farbpalette, Typografie, Abstände, Komponenten.
   - Keine neuen UI-Varianten ohne Abgleich mit bestehendem Design-System.

2. **UI/UX-Qualität**
   - Klare Informationshierarchie (Was esse ich heute? Wie koche ich das?).
   - Schnelle Nutzbarkeit: max. 2 Klicks vom Tagesmenü zur Kochanleitung.
   - Lesbarkeit bei Rezepten (Zutatenliste + Schritt-für-Schritt).

3. **Responsive Design**
   - Mobile-first, Desktop-optimiert.
   - Breakpoints prüfen (>=360px, 768px, 1024px, 1440px).
   - Navigation, Karten, Listen und Buttons auf Touch-Geräten gut bedienbar.

4. **Accessibility (Basisstandard)**
   - Sichtbare Fokuszustände, ausreichende Kontraste, semantische HTML-Struktur.
   - Kein informationskritischer Inhalt nur über Farbe codiert.

5. **SEO-UI-Signale**
   - Sprechende URL-Struktur beibehalten.
   - Interne Verlinkung zwischen Menü, Archiv und Rezeptseiten ausbauen.
   - Konsistente Seitentitel + Meta-Beschreibungen unterstützen.

## Release-Gate (Designer-Checkliste)
- [ ] Brand-Styles konsistent auf allen Kernseiten
- [ ] Mobile (360px/390px) ohne horizontales Scrollen
- [ ] Desktop (>=1024px) mit klarer 2-Spalten-Struktur, wo sinnvoll
- [ ] Rezeptseiten vollständig lesbar (Zutaten + Schritte)
- [ ] Navigation Archiv ↔ Tagesmenü ↔ Rezept funktioniert
- [ ] Accessibility-Basics erfüllt (Kontrast, Fokus, Semantik)

## KPI-Empfehlungen für Product-Market-Fit
- Time-to-Recipe (Startseite -> gewünschtes Rezept)
- Scroll-Tiefe auf Rezeptseiten
- Anteil Nutzer, die Archiv nutzen
- Rückkehrquote (7/30 Tage)
