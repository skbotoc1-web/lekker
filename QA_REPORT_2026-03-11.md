# QA Report – Senior Tester (2026-03-11)

## Scope
- Smoke Frontend (/, /menue, /menue/:day, /rezept/:option/:day/:slot)
- Smoke Backend (/health, /api/menu/today, /feeds/menu-today.json)
- Data integrity check (menus vs recipes)

## Findings

### 1) Broken recipe links on homepage (High)
**Observed**
- `/` shows menu for `2026-03-12` (status draft).
- Clicking recipe link leads to `Rezept nicht gefunden.` (404 body).

**Repro**
1. Open `http://localhost:8787/`
2. Click any `Rezept` link
3. See error text `Rezept nicht gefunden.`

**Technical evidence**
- DB has menu `2026-03-12` with **0 recipes**.
- DB has menu `2026-03-11` with **10 recipes**.

**Expected**
- Homepage must not show recipe links for menus without recipes.

**Fix recommendation**
- In `/` route, select latest menu with recipe completeness (>=10 expected rows or strict slot coverage).
- Alternative: if incomplete, display clear "in Vorbereitung" state and hide recipe links.

---

### 2) Inconsistent "today" behavior between `/` and API (Medium)
**Observed**
- `/api/menu/today` returns `2026-03-11`.
- `/` renders `2026-03-12` (latest by day DESC).

**Expected**
- Consistent rule: either "today" only, or "latest published/complete".

**Fix recommendation**
- Align route logic:
  - `/` should use `day = getDayToday()` OR
  - `/` should use latest *complete* menu and label it as "Neuester Menüvorschlag" (not "Tagesmenü").

---

### 3) Data integrity gap in pipeline output (High)
**Observed**
- Menu row can exist without corresponding recipes.

**Risk**
- Broken UX, dead links, reduced trust.

**Fix recommendation**
- Make pipeline write atomic per day (transaction):
  1. create/replace menu
  2. generate all recipe rows
  3. commit only when complete
- Add validation gate: fail stage if recipe count != expected slots.

## Additional cycle (18:27 UTC)

### 4) Pipeline creates future "draft" menus without recipes (Critical)
**Observed**
- DB now contains `2026-03-12` and `2026-03-13` with **0 recipes** each.
- Homepage always picks latest day (`ORDER BY day DESC`) and exposes recipe links that fail.
- `/api/menu/today` still serves `2026-03-11` with 10 recipes.

**Impact**
- Severe user confusion (UI says "Tagesmenü", data is not actionable).
- Recurring broken-link experience.

**Fix recommendation**
- Enforce completeness before menu becomes selectable by UI.
- Add DB integrity rule/check: for selectable menu require expected recipe cardinality.
- Prevent creation of date records without recipes, or mark as hidden/prep.
